WeClone-用微信聊天记录打造自己的AI数字分身

WeClone：让你的聊天记录拥有“灵魂”

你是否曾想过，如果你的聊天风格、你的口头禅、你独特的表达方式，能够被一个 AI 所学习，创造出一个数字世界里的“你”？xming521/WeClone 项目 (欢迎访问 GitHub - xming521/WeClone 并给它一个 Star ⭐) 就是这样一个激动人心的尝试。它是一个一站式解决方案，旨在通过分析你的微信聊天记录来微调大语言模型（LLM），让模型说话有“你那味儿”，并能将其绑定到聊天机器人，实现你专属的数字分身。

这个项目不仅涵盖了从聊天数据处理到模型微调的全过程，甚至还包括了声音克隆的探索，目标是让你的数字分身不仅“言如其人”，更能“声似其人”。

本篇博客将作为一份详细的指南，重点介绍 Windows 用户 如何一步步上手 WeClone 项目，打造属于自己的数字克隆。

⚠️ 重要提示： WeClone 项目仍在快速迭代中，当前效果不代表最终效果。微调 LLM 的效果很大程度上取决于模型大小、聊天数据的数量和质量。理论上，模型越大，数据越多，效果越好。同时，官方明确指出 Windows 环境未进行严格测试，强烈建议使用 WSL (Windows Subsystem for Linux) 作为运行环境。

在开始之前，请务必注意：保护个人隐私至关重要，请勿泄露任何个人信息！同时，请勿将此项目用于任何非法用途，否则后果自负。

准备工作：搭建你的“克隆工厂”

要在 Windows 上顺利运行 WeClone，我们需要先搭建好相应的环境。

1. Windows Subsystem for Linux (WSL) - 强烈推荐

鉴于官方的建议和 Python 项目在 Windows 上的普遍兼容性挑战，使用 WSL 是最稳妥、最高效的选择。

  • 安装 WSL：
    • 打开 PowerShell (管理员权限)，输入 wsl --install。这将安装默认的 Ubuntu 发行版。
    • 如果你需要其他 Linux 发行版，可以使用 wsl --list --online 查看可用列表，并用 wsl --install -d <DistroName> 安装。
    • 安装完成后，重启电脑。
    • 你可以在 Microsoft Store 中找到并安装你选择的 Linux 发行版。
  • 访问 WSL： 安装完成后，你可以在开始菜单找到你的 Linux 发行版并启动它，就像打开一个普通的应用程序一样。

2. Python 环境

WeClone 项目需要 Python 3.10 或更高版本。

  • 在 WSL 中安装 Python：

    • 打开你的 WSL 终端。
    • 更新包列表：sudo apt update && sudo apt upgrade
    • 安装 Python 3.10 (以 Ubuntu 为例)：
    • 验证安装：python3.10 --version 和 pip3 --version。
    • 建议创建一个虚拟环境来管理项目依赖： 之后所有的 Python 相关操作都在激活的虚拟环境中进行。

  • (不推荐) 直接在 Windows 中安装 Python：

    • 从 Python 官网 (python.org) 下载 Python 3.10+ 的 Windows 安装包。
    • 安装时，务必勾选 “Add Python to PATH”。
    • 同样建议使用虚拟环境：python -m venv weclone_env 然后 .\weclone_env\Scripts\activate。

3. Git 版本控制

你需要 Git 来克隆 WeClone 项目仓库。

  • 在 WSL 中安装 Git： sudo apt install git
  • (Windows) 安装 Git： 从 Git for Windows 下载并安装。

4. (可选) NVIDIA 显卡驱动和 CUDA (针对 GPU 微调)

如果你计划使用 NVIDIA GPU 进行模型微调（推荐，速度更快），你需要：

  • 在 Windows 上安装最新的 NVIDIA 显卡驱动。
  • 在 WSL 中安装 CUDA Toolkit：
    • 确保你的 Windows NVIDIA 驱动版本支持你希望在 WSL 中安装的 CUDA 版本。
    • 访问 NVIDIA 官网下载适用于 WSL-Ubuntu 的 CUDA Toolkit。按照官方指南进行安装。这一步可能较为复杂，请务必仔细阅读 NVIDIA 的官方文档。

第三步：准备你的微信聊天记录

这是创造你“数字分身”的核心“原料”。

  1. 导出微信聊天记录：
    • 你需要将微信聊天记录导出为 .txt 或 .csv 文件。具体导出方法因微信版本和工具而异，你可能需要借助第三方工具或脚本。请确保导出的是纯文本对话内容。
    • 注意隐私！ 导出的数据包含你的个人对话，请妥善保管。
  2. 数据格式化：
    • WeClone 通常需要特定格式的对话数据。最常见的是包含至少两列的 CSV 文件，例如 speaker (说话人) 和 text (内容)。
    • 项目内可能会提供数据处理脚本或示例 (example_chat.csv)，请仔细阅读项目 README 中关于数据准备的部分。
    • 根据项目最新说明 (0.2.0 版本后)，csv 文件夹需要放在 dataset 目录下。你需要创建 WeClone/dataset/csv/ 这样的目录结构，并将你的聊天记录 CSV 文件放入其中。
  3. 数据清洗 (重要)：
    • 高质量的数据是微调成功的关键。删除不相关的自动回复、系统消息、广告等。
    • 确保对话连贯且能真实反映你的语言风格。
    • 项目本身也可能提供数据清洗相关的脚本或建议。

第四步：配置参数 (settings.jsonc)

项目的核心配置在 settings.jsonc (或者早期版本的 settings.json) 文件中。你需要根据你的需求和硬件条件修改此文件。

打开 WeClone/settings.jsonc (如果不存在，可能是 settings.templ 或类似名称的模板文件，你需要复制并重命名为 settings.jsonc)，主要关注以下参数：

  • 模型选择 (model_name_or_path, template)：
    • model_name_or_path: 指定基础语言模型。可以是 Hugging Face Hub 上的模型名称 (如默认的 Qwen/Qwen2.5-7B-Instruct) 或本地模型路径。
    • template: 对话模板，需要与所选模型匹配。
  • 数据集路径 (dataset_dir, dataset)：
    • dataset_dir: 通常指向包含你聊天记录 csv 文件夹的 dataset 目录。
    • dataset: 你在 csv 文件夹中准备的数据集文件名 (不含扩展名)。
  • 训练参数：
    • per_device_train_batch_size: 每个设备的训练批量大小。根据你的显存大小调整，显存不足时减小此值。
    • gradient_accumulation_steps: 梯度累积步数。在显存不足时，可以通过增大此值来模拟更大的批量，但会增加训练时间。
    • learning_rate: 学习率。
    • num_train_epochs: 训练轮次。
    • lora_target: LoRA 微调的目标模块，不同模型可能有所不同。
    • output_dir: 微调后模型的保存路径。
  • 推理参数 (infer_args)：
    • max_new_tokens: 生成回复的最大长度。
    • temperature: 控制生成文本的随机性，值越高越随机。
    • top_p, top_k: 控制采样策略。
  • 系统提示 (default_system)：
    • 微调时使用的默认系统提示词。这个很重要，后续在聊天机器人中设置系统提示词时需要与此一致，才能获得最佳效果。
  • 其他：
    • 是否使用 flash_attention (如果硬件支持，可以加速训练)。
    • GPU 设置 (ddp_timeout, deepspeed 配置等，如果使用多卡训练)。

请仔细阅读 settings.jsonc 文件中的注释和项目 README 中的参数说明，根据你的实际情况进行调整。

第六步：部署和使用你的数字分身

WeClone 项目通常推荐使用 AstrBot (GitHub - AstralVC/AstrBot) 作为聊天机器人平台来部署你微调后的模型。

  1. 部署 AstrBot： 参考 AstrBot 的官方文档进行安装和配置。

  2. 启动 API 服务：

    WeClone 提供了一个 API 服务，供 AstrBot 调用你微调后的模型。

    Bash

    你需要确保这个 API 服务正在运行，并且 AstrBot 配置为连接到此服务。

  3. 配置 AstrBot：

    • 在 AstrBot 中设置模型为你微调后的模型路径或 API 地址。
    • 重要： 在 AstrBot 中设置系统提示词 (System Prompt)，使其与你微调时在 settings.jsonc 中设置的 default_system 完全一致。
    • 关闭工具调用： 微调后的模型通常专注于模仿你的语言风格，而不是执行工具调用。在 AstrBot 中向你的机器人发送指令 /tool off all 来关闭所有默认工具，否则可能无法看到微调效果。

  4. 开始聊天： 现在，你可以和你的“数字分身”聊天了！看看它是否真的有“你那味儿”了。

第七步：使用 weclone-cli (命令行工具)

WeClone v0.2.1 版本后引入了 weclone-cli 命令行工具，它简化了项目的某些操作。

  • 安装后 (通过 uv pip install -e .)，你可以通过 weclone-cli --help 查看所有可用的命令和选项。
  • 常用功能可能包括数据处理、训练、启动 API 服务等。

Windows 原生环境可能遇到的问题 (及 WSL 的优势)

如果你坚持不使用 WSL 而直接在 Windows 上运行：

  • 依赖安装： 很多 Python 包，尤其是涉及 C++ 编译的（如 bitsandbytes, flash-attention 等），在 Windows 上安装可能会非常棘手，需要 Visual Studio C++ 构建工具和各种库，且容易出错。WSL 中这些通常更容易通过包管理器解决。
  • 文件路径： Windows 的路径分隔符是 \，而 Linux/macOS 是 /。代码中如果路径处理不当，在 Windows 上会出问题。
  • 长文件名/路径限制： Windows 对文件路径长度有限制，深度嵌套的项目或数据集可能超限。
  • CUDA 和 PyTorch： 在 Windows 上配置 PyTorch GPU 版本和 CUDA 有时比在 Linux 环境中更复杂，驱动和库版本兼容性问题更多。
  • Shell 脚本： 项目中如果包含 .sh 脚本，在 Windows CMD 或 PowerShell 中无法直接运行，需要 Git Bash 或手动转换为批处理/PowerShell 脚本。
  • 性能： 对于计算密集型任务，WSL2 提供了接近原生的 Linux 性能，通常优于直接在 Windows 上运行某些类型的应用。

因此，再次强调，使用 WSL 是在 Windows 上运行 WeClone 的首选方案。

总结与展望

通过 WeClone 项目，我们得以一窥“数字永生”的未来。虽然目前技术仍在发展，效果也依赖诸多因素，但这无疑是一个充满想象力和潜力的方向。

希望本篇 Windows (及 WSL) 环境下的 WeClone 使用指南能帮助你顺利开启你的“数字分身”打造之旅。记住，耐心、细致的配置和高质量的数据是成功的关键。

如果你在实践中遇到问题，可以：

  • 仔细阅读 WeClone 项目的 GitHub README 和 Issues。
  • 向项目社区提问 (在 GitHub Discussions 或 Issues 中)。

最后，如果你觉得 WeClone 项目对你有帮助，别忘了给它一个 Star ⭐，支持开发者的辛勤付出！
